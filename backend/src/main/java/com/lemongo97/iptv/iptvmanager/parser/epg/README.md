## 什么是 XMLTV DTD？

XMLTV DTD 是一套用来定义电视节目表数据结构的规则。这个格式不会把节目按照日期或者频道进行生硬的分组。相反，节目的播出时间和频道都是作为 `programme`（节目）元素的属性来记录的。

### 核心设计特点

* **支持多语言**：许多元素都有 `lang` 属性。你可以在同一个文件里使用不同的语言，或者为同一个节目提供多种语言的描述。
* **数据格式宽松**：这个文件格式里绝大多数的元素都是可选的。你可以根据需要，写出非常简单的节目表文件。

---

## 全局时间与文本格式规范

### 时间格式

文件里所有的时间都使用一种基于 ISO 8601 的格式。

* 完整格式：`YYYYMMDDhhmmss +时区`（例如：`19880523083000 +0300`）。
* 模糊格式：如果你只知道年份和月份，可以只写 `YYYYMM`（例如：`200209`）。
* 时区：如果不对时区进行标注，系统会默认时间为 UTC 时间。

### 文本限制

* 除非特别说明，否则文本元素内部**不能包含换行符**。这是为了方便人们把数据转成单行文本格式。
* 文本开头和结尾的空格没有实际意义，系统会自动忽略。

## 核心元素详细说明

### 1. 根元素 `<tv>`

这是整个 XML 文件的最外层标签，用来包裹所有的频道和节目信息。

* **可用属性**：
* `date`：这份节目表最初生成的日期。
* `source-info-name`：数据提供方的名字。
* `source-info-url`：数据提供方的网站链接。
* `source-data-url`：获取原始数据文件的链接。
* `generator-info-name`：生成这份文件的软件名字和版本号。
* `generator-info-url`：该软件的官方网址。



### 2. 频道元素 `<channel>`

用来记录频道的详细信息。节目元素会通过 `id` 属性与这里定义的频道进行关联。

* **内部子元素**：
* `<display-name>`（必须有至少一个）：频道的名称或频道号。最常用的名字应该排在最前面。
* `<icon>`（可选）：频道的图标。包含图片地址（`src`）、宽度（`width`）和高度（`height`）。
* `<url>`（可选）：频道的官方网站或粉丝页面。


* **属性**：
* `id`（必须有）：频道的唯一标识符。建议使用类似不带协议的域名格式（例如：`channel-one.tv`）。



### 3. 节目元素 `<programme>`

用来记录某一个具体节目的播出信息。

* **核心属性**：
* `start`（必须有）：节目开始时间。
* `stop`（可选）：节目结束时间。节目表默认在前一个节目结束时，后一个节目立刻开始，中间没有缝隙。
* `channel`（必须有）：播出该节目的频道 `id`。
* `clumpidx`（默认值为 `0/1`）：用来解决同一个时间段播出多个节目（例如“新闻与天气”）的冲突问题。


* **常用内部子元素**：
* `<title>`（必须有）：节目名称，例如《辛普森一家》。
* `<sub-title>`（可选）：副标题或者单集名称。
* `<desc>`（可选）：节目简介。这个元素内部允许使用换行符。
* `<credits>`（可选）：演职人员名单。包含导演（`director`）、演员（`actor`，可标注角色和是否为客串）、编剧（`writer`）和主持人（`presenter`）等。
* `<date>`（可选）：节目制作完成的日期或版权日期。
* `<category>`（可选）：节目类型，例如“喜剧”或“悬疑”。一个节目可以有多个类型。
* `<keyword>`（可选）：节目的关键词。多个单词的词组建议用连字符连接（例如 `super-hero`）。
* `<language>` / `<orig-language>`（可选）：播出的语言以及原始语言（适用于配音节目）。
* `<length>`（必须指定单位）：节目的实际时长，不包含广告和预告片。
* `<country>`（可选）：制作国家，可以使用国家名称或两位大写国家代码（如 `US`、`GB`）。

## 高级与特有元素规范

### 1. 集数系统 `<episode-num>`

用来记录节目的季数和集数。通过 `system` 属性来区分不同的计数方式：

* **xmltv_ns 系统**：
  使用点号分隔的三个数字（`季.集.篇幅`），所有数字都**从 0 开始计算**。
* 例如 `1.0.0/1` 表示：第二季的第一集。
* 如果是上下集，上集写 `1.0.0/2`，下集写 `1.0.1/2`。
* 如果只知道是第一季，可以写成 `0..`。


* **onscreen 系统**（默认值）：
  直接抄录电视里播出的单集标签，例如 `Episode #FFEE`。
* **第三方数据库系统**（2013年引入）：
  支持直接引用 `themoviedb.org`、`thetvdb.com` 或 `imdb.com` 的数据编号。

### 2. 视频与音频配置

* **`<video>`**：
* `<present>`：是否有画面（`yes` 或 `no`）。如果是 `no`，说明这是广播电台。
* `<colour>`：是否为彩色（`yes` 或 `no`）。
* `<aspect>`：画面比例，例如 `16:9`。
* `<quality>`：画面画质，例如 `HDTV`。


* **`<audio>`**：
* `<present>`：是否有声音。
* `<stereo>`：声音声道类型，例如 `mono`（单声道）、`stereo`（双声道）、`surround`（环绕声）或 `bilingual`（双语独立声道）。



### 3. 特殊播放标签

* **`<previously-shown>`**：表示该节目是重播。可以带有之前的播出时间和频道属性。
* **`<premiere>`**：首播标签。可以在标签内写明具体含义（例如：在该频道第一次播出）。
* **`<last-chance>`**：最后一次播放标签。通常用于电视台购买播放权后，最后一次播放该电影。
* **`<new>`**：全新节目标签。指这个节目从来没有在任何地方或者该国家播放过。第二季或者老节目的新一集不能使用这个标签。
* **`<subtitles>`**：字幕信息。属性 `type` 可以是 `teletext`（数字文字字幕）、`onscreen`（内嵌硬字幕）或 `deaf-signed`（手语翻译）。

### 4. 评分与评论

* **`<rating>`**：分级控制。例如使用 `MPAA` 系统，评分为 `NC-17`。
* **`<star-rating>`**：星级评价。格式必须是 `N / M`（例如 `4/5` 表示五星里给四星）。
* **`<review>`**：节目评论。属性 `type` 必须是 `text`（文本内容）或者 `url`（评论链接）。可以标注评论来源（`source`）和评论人（`reviewer`）。

### 5. 图片元素 `<image>`

用来为节目或演职人员提供图片。

* 节目图片类型：`poster`（海报）、`backdrop`（背景图）或 `still`（节目截图）。
* 人员图片类型：`person`（个人艺术照）或 `character`（剧照）。
* 属性：`size`（1 表示小于 200px，2 表示 200-400px，3 表示大于 400px），`orient`（P 表示竖屏，L 表示横屏）。
